Key Differences: ESME vs SMSC Binding

Aspect ESME Bind SMSC Bind
Purpose Client connecting to carrier/gateway Peer-to-peer SMSC routing
Common Use Application sending SMS SMSC-to-SMSC forwarding
Authentication Usually system_id + password Often IP-based + credentials
Typical Role ESME always client Either can be client/server
Traffic Pattern App → Gateway → Carrier Gateway ↔ Gateway

Connection Initialization

Connection Who Initiates TCP Who Sends Bind Who Listens
Server 1 → Server 2 Server 1 (client) Server 1 Server 2 (server)
Server 2 → Server 1 Server 2 (client) Server 2 Server 1 (server)
Note: The role (ESME vs SMSC) is logical and declared in the bind command, but the TCP connection mechanics remain the same - client initiates, server listens.
